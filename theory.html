Normality Borel, 1909

Write a real number \(\omega\) in base \(b\). It is simply \(b\)-normal if each of the \(b\) digits occurs with density \(1/b\); it is \(b\)-normal if for every \(n\) each of the \(b^{n}\) strings of length \(n\) occurs with density \(b^{-n}\); and it is absolutely normal if it is \(b\)-normal for every base \(b \geq 2\).

Borel proved that almost every real number is absolutely normal: the exceptions form a set of Lebesgue measure zero. The proof is a Borel–Cantelli argument and exhibits nothing. More than a century later the situation is still the one that makes the subject peculiar — normality is generic, easy to prove for almost all numbers, and impossible to establish for any number anyone cares about. Nobody knows whether \(\pi\), \(e\), \(\sqrt{2}\) or \(\log 2\) is normal in a single base.

Explicit normal numbers do exist, and they are built rather than found. Champernowne's constant, the positive integers written out one after another,

\[ C_{10} \;=\; 0.1234567891011121314\ldots \]

is \(10\)-normal, and the same construction is \(b\)-normal in every base (Nakai–Shiokawa). Sierpiński produced an absolutely normal number in 1917; whether it could actually be computed had to wait until Becher and Figueira in 2002.

Density need not exist

Before asking whether a density equals \(1/b\), one has to ask whether it exists. It need not. The number

\[ \omega \;=\; \sum_{i=0}^{\infty}\left(\sum_{j=0}^{2^{2i}-1} b^{-j}\right) b^{-2^{2i}} \;=\; 0.\underbrace{1}_{1}\underbrace{00}_{2}\underbrace{1111}_{4}\underbrace{0\cdots0}_{8}\ldots \]

has no density of zeros at all: the running frequency keeps oscillating for ever. Numbers for which every string of every length does have a density are the Borel numbers, written \(\mathcal{B}\), and everything below lives inside that class.

Normality as independence

For a Borel number the digit frequencies are an honest probability, and normality turns into a statement about randomness that is exactly what the intuition promises.

Theorem

A Borel number \(\omega\) is \(b\)-normal if and only if it is simply \(b\)-normal and its digits are mutually stochastically independent:

\[ \mathbb{P}\big([s_1,\ldots,s_n]\big) \;=\; \mathbb{P}\big([s_1]\big)\cdots\mathbb{P}\big([s_n]\big) \;=\; \frac{1}{b^{\,n}} . \]

So the gap between simple normality and normality is precisely independence, and neither half implies the other. Independence alone is not enough — the digits of an integer are trivially independent, being all zero — and equidistribution alone is not enough either, as the Thue–Morse digits show: they are half zeros and half ones, and yet the string \(000\) never occurs.

Equivalently, \(\omega\) is \(b\)-normal exactly when it is casually constructed: the first digit is uniform and every digit is uniform given all the digits before it,

\[ \mathbb{P}\big([s_1]\big) = \frac{1}{b}, \qquad \mathbb{P}\big([s_n] \mid [s_1,\ldots,s_{n-1}]\big) = \frac{1}{b}. \]

A number generated by rolling a fair \(b\)-sided die is normal; one generated by a loaded die is not.

The swap map \(\sigma^{i,j}_{(b)}\)

Fix a base \(b\) and two digits \(i, j\). The swap map \(\sigma^{i,j}_{(b)}\) exchanges them everywhere in the base-\(b\) expansion of its argument, and the object of interest is the difference it produces:

\[ \Delta_b^{\,i,j}(\omega) \;=\; \omega - \sigma^{i,j}_{(b)}(\omega). \]

For example, on the first twenty decimal digits of \(\pi\),

\[ \begin{aligned} \pi &= 3.14159265358979323846\ldots \\ \sigma^{1,5}_{(10)}(\pi) &= 3.54519261318979323846\ldots \end{aligned} \]

The two numbers have the same digits in a different arrangement, so their difference could a priori be anything at all. It cannot.

Lemma — only six digits survive

Let \(\delta = |i-j|\). Whatever \(\omega\) is, every digit of \(\Delta_b^{\,i,j}(\omega)\) lies in

\[ \mathfrak{D} \;=\; \{\, \delta,\; b-\delta,\; \delta-1,\; b-\delta-1,\; b-1,\; 0 \,\} \pmod b . \]

The reason is the arithmetic of borrows. The two expansions differ only at the positions carrying \(i\) or \(j\); those contribute \(\pm\delta\), and each negative contribution generates a borrow which travels leftwards until it meets a position of the other kind. A position of \(i\) yields \(\delta\) or \(\delta-1\), a position of \(j\) yields \(b-\delta\) or \(b-\delta-1\), and every position in between yields \(0\) or \(b-1\) according to whether a borrow is passing through it.

This is what the strip on the calculator page shows: two rows of full-alphabet noise, and underneath a row drawn from a handful of symbols.

The scheme \(\Delta_b\)

The \(\Delta_b\)-scheme of a number is the table of the densities of those digits, one line for each pair \(i \neq j\). If \(\omega\) is normal the borrows behave like coin tosses, and the whole table is forced.

Proposition — every normal number has the same scheme

If \(\omega\) is \(b\)-normal then its \(\Delta_b\)-scheme depends on \(i\) and \(j\) only through \(\delta = |i-j|\), and each line reads

\[ \mathbb{P}[\delta] = \mathbb{P}[b-\delta] = \mathbb{P}[\delta-1] = \mathbb{P}[b-\delta-1] = \frac{1}{2b}, \qquad \mathbb{P}[b-1] = \mathbb{P}[0] = \frac{b-2}{2b}, \]

where the probabilities of digits that coincide modulo \(b\) are added together.

Where the numbers come from: half the swapped positions carry \(i\) and half carry \(j\), and independently of that a borrow arrives half the time, which splits the density \(2/b\) of the swapped positions four ways and gives \(1/(2b)\) each. The remaining positions, of density \(1 - 2/b\), split evenly between \(0\) and \(b-1\).

A number whose scheme is this one is called \(b\)-pseudonormal, and the set of them is written \(\mathcal{D}_b\). Since the table has \(b(b-1)/2\) lines and each entry is a finite count, the condition can actually be tested — which normality cannot.

Where pseudonormality sits

For every base \(b \neq 3\), and for Borel numbers,

\[ \omega \ \text{casually constructed} \iff \omega \in \mathcal{N}_b \implies \omega \in \mathcal{D}_b \implies \omega \in \mathcal{S}_b , \]

and both implications are strict:

\[ \mathcal{N}_b \subsetneq \mathcal{D}_b \subsetneq \mathcal{S}_b \subsetneq \mathcal{B} \subsetneq \mathbb{R} . \]

Strictness is not abstract. The rational number

\[ \frac{7930067}{16843009} \;=\; 0.\overline{1320201331020231}_{\,4} \]

has exactly the prescribed \(\Delta_4\)-scheme, and yet no rational is normal in any base: pseudonormal does not imply normal. In the other direction the periodic base-4 number \(0.\overline{33323130232221201312111003020100}\) is simply \(4\)-normal — it contains each digit, and indeed each pair of digits, equally often — but its lines for \((0,1)\) and \((1,2)\) disagree, so it is not \(4\)-pseudonormal.

Base 3 is an exception

At \(b = 3\) the six admissible digits collapse onto one another for both values of \(\delta\), and the scheme degenerates into the single demand that the three digits of \(\Delta\) be equally likely. That is genuinely weaker than simple normality: the rational

\[ \frac{3617}{265720} \;=\; 0.\overline{000100220221}_{\,3} \]

is exactly \(3\)-pseudonormal while its digits have densities \(\tfrac{1}{2}, \tfrac{1}{6}, \tfrac{1}{3}\). In base 3, and only there, the two conditions are not comparable.

Base 2 is degenerate

At \(b = 2\) the swap complements every digit, so \(\Delta\) is the expansion itself shifted by one place — possibly complemented — and its two densities are those of \(\omega\) in some order. Pseudonormality and simple normality then say exactly the same thing, and the calculator returns the same verdict for both. The Thue–Morse digits are the specimen to try: simply \(2\)-normal, \(2\)-pseudonormal, and not \(2\)-normal.

All bases at once

One base is not enough to recover normality. All the powers of one base are.

Theorem

A Borel number \(\omega\) is \(b\)-normal if and only if it is \(b^{n}\)-pseudonormal for every \(n \geq 2\). Consequently

\[ \omega \ \text{absolutely normal} \iff \omega \in \mathcal{D}_b \ \text{ for every } b \in \mathbb{N}_{>1} . \]

The argument is short: pseudonormality in a base \(b^{n} \geq 4\) forces simple normality in that base, and by Pillai's classical characterisation simple normality in every base \(b^{n}\) is the same thing as normality in base \(b\). The exceptional base 3 never appears, since \(b^{n} \geq 4\) as soon as \(n \geq 2\).

So pseudonormality is strictly weaker than normality in each single base and exactly equivalent to it across all of them. What remains open is the quantitative question, which is the one that matters if the test is to be used in practice: is there a finite set of bases whose schemes already force normality, or can a non-normal number fake the first \(N\) of them for every \(N\)?
