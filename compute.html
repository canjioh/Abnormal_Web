Strictness is the tolerance of the verdict, in standard errors (σ). On a finite prefix each density is only measured, so it never lands exactly on the prescribed value; the run is called pseudonormal when no entry sits further than this many σ from where it should be. Lower is more forgiving, higher is more suspicious. It only affects measured (non-periodic) numbers — for a rational the whole period is known and the check is exact, so the slider does nothing.

Verdict

This is a test of pseudonormality, not of normality. Every normal number is pseudonormal, so a no is a proof that the number is not normal; a yes is necessary but not sufficient — some rationals, normal in no base at all, pass it.

The two digits being exchanged are set in vermilion. The third row is the difference: it draws on at most six digits, whatever the number and whatever the base, and the rare ones — the digits other than \(0\) and \(b-1\) — are picked out in indigo.

The scheme \(\Delta_b\) — one line per pair

above the prescribed value ▲ below the prescribed value ▼ intensity ∝ standard errors

A number is pseudonormal when every line matches the prescribed one and lines with equal \(\delta\) agree with each other. Failing the second while passing the first is possible, and is how simply normal numbers that are not pseudonormal get caught.
